RESET AI · Design direction · v1

Calm confidence, delivered like hospitality.

The user arrives anxious about a text. Every surface borrows Airbnb's visual manners — warm white, one coral accent, soft-shadow cards, generous air — so the app feels like a trusted concierge, not a hype machine. The theater is saved for one moment: the debate reveal.

Point of view

Four principles that decide every screen

From the Hook blueprint

The dropzone is the app

Home is one giant, inviting upload surface. No onboarding carousel, no category picker. One tap opens the photo library. Everything else on Home is skippable.

Airbnb DNA

Quiet chrome, warm cards

White ground, hairline dividers, 14–16px radii, soft diffuse shadows. Exactly one accent — Rausch coral — reserved for the primary action and the gauge. Never two coral elements competing on one screen.

Variable reward

Theater in one place only

The debate reveal is the show: staggered agent cards, skeleton-to-take resolution, a paced 1.2–1.8s dwell, haptic ticks, then the gauge locks with a success haptic. Every other screen stays still so this one feels alive.

Content boundary

Warm, never smug

Tone of every string: a wise friend, win-win framing. No contempt, no manipulation aesthetics — visually that means no "dark pattern" reds, no aggressive ALL-CAPS urgency, no fake scarcity.

Foundations

Color, type, shape

The palette is Airbnb's own lineage: Rausch as the single product accent, plus one hue per persona so the debate reads at a glance — Arthur is composed teal, Clara is empathic plum, Leo is playful orange.

Shape, elevation, motion

  • radius/card14px cards · 12px buttons · 999px chips & avatars
  • space4pt grid · 20px screen gutters · 24px between card groups
  • elevationOne shadow only: 0 1px 2px 6% + 0 6px 20px 7% — never stacked
  • hairline1px #E8E5E6 for list dividers; borders on cards that sit on white
  • motionSpring (damping .8) for card entrances; 300ms ease-out for everything else; reduced-motion honored
  • hapticsLight impact per agent card · Success notification at gauge lock (required, per blueprint)

Straight from the blueprint's mechanics table — these are contract, not garnish. Build order for engineering is included.

Handoff

What gets built, where

Every screen above maps 1:1 onto files already scaffolded in mobile/ — this is a re-skin + choreography pass, not a restructure. Proposed order:

Step 1

Design tokens

New src/theme/tokens.ts: the palette, type scale, radii, and shadow above as typed constants. Every component below consumes only tokens.

Step 2

Home + tab bar

app/index.tsx + ImagePickerDropzone restyled; zero-friction path verified against a device.

Step 3

Debate reveal

Persona colors, edge-accent cards, dwell-floor pacing + haptic ticks in debate-reveal.tsx, driven by mock SSE events first (mock-first rule).

Step 4

Result + gauge

AttractionGauge spring/overshoot + success haptic; result layout mirroring SynthesisResult.

Step 5

Contacts + story card

Memory list with loss-aversion banner; gradient export card via StoryCardExport.

Later

Dark theme

Tokens make it a palette swap. Not in v1 scope — Airbnb itself shipped light-only for a decade.
