Design spec · v2.1 · Dusk direction · complete pass

Confidence has a golden hour.

A warmer, glass-and-gradient reinterpretation of RESET AI — built around the five minutes before you walk in, not just the debate over what to text. This pass carries every screen from v1 forward into the dusk system: the live Arthur/Clara/Leo debate, the result screen, saved contacts, and story sharing, alongside the new Dashboard, Vibe Check-in, Live Scenario, Recap, and Ranking. Kept alongside ui-spec-v1.html (the Airbnb-inspired direction) for side-by-side comparison, not as its replacement.

Principles

Four calls that decide every screen

Where v1 borrowed hospitality-brand calm, v2 borrows the warmth of a friend hyping you up in the bathroom mirror before you walk back out.

01 · Tone

Warm before smart

Frosted glass, gradient light, and a rounded display face read as encouragement first. The analysis is still rigorous underneath — it just doesn't lead.

02 · Color

One mode, everywhere

Whichever Social Mode you're in — Hype, Chill, Romantic, Direct — tints the meter, the buttons, and the active cards for that whole session. The app visibly matches your energy.

03 · Advice

Momentum you can check off

Live coaching reads as a short list of physical next actions — Approach, Open, Build Rapport, Exit warm — not paragraphs to read mid-conversation.

04 · Memory

The night remembers you

Recap and Ranking turn one night out into a running story with friends, not a one-off score you forget by morning.

Foundations

Color, type, glass

A dusk palette — golden-hour warmth cooling into night — instead of v1's daylight neutrals. Backgrounds are warm cream, never pastel blue; every interactive surface is frosted glass over a soft gradient, never flat.

Agent identity (Arthur/Clara/Leo) and Social Mode color are two separate systems, kept deliberately apart: Social Mode tints whichever screen you're on for the whole session; Agent identity is scoped to the debate and result screens only, so a card is never ambiguous about whether it's "your mode" or "who's talking."

Display face is the ui-rounded CSS generic (SF Pro Rounded on Apple platforms) — production ships Aeonik, licensed and self-hosted at build time. Body copy stays on the plain system stack for maximum legibility at small sizes.

Shape & glass

  • Card radius 20–28px, never sharp
  • Pill radius 999px — buttons, tabs, chips, tab bar
  • Glass recipe rgba(255,255,255,.6) + blur(14–18px) + 1px rgba(255,255,255,.7) border
  • Shadow warm-tinted, never pure grey — shadow color borrows the accent
  • Elevation floating cards overlap the hero banner by 24–40px

Motion & haptic choreography

Motion is warmer and bouncier than v1's restrained hospitality easing — springs, not linear fades — but every rule below still respects prefers-reduced-motion.

MomentMotionHaptic
Dashboard loadMode card and Playbook cards stagger in, 60ms apart, spring (damping 18)—
Blob mode selectTapped blob scales 1 → 1.12 → 1 with a squash-stretch wobbleLight impact
Confidence Meter fillTrack fills with a spring, 900ms; number counts up in stepLight impact per +10, success on full
Step check-offDot morphs circle → check with a 200ms scale pop; next step's ring pulses onceMedium impact
Playbook horizontal scrollSnap-to-card, momentum-based, no forced animation—
Recap openStat numbers count up from 0 over 700ms, staggered 80ms per tile—
Podium revealBars rise from 0 height, 3rd → 2nd → 1st, 120ms staggerSuccess on 1st place landing
Handoff

What gets built, where

Component vocabulary only — no implementation yet. Confirm this direction (or request changes) before any Next.js/Tailwind/Framer Motion code is written, per the brief.

Component

VibeSelector

The four-blob mode picker. Used in the Vibe Check-in modal and again as a compact chip in the Dashboard header.

Component

PlaybookCard

Gradient-thumbnail scenario card in the horizontal Playbook feed, filterable by venue-type pill.

Component

GradientMeter

The Confidence Meter — animated gradient progress track plus a large tabular-numeral readout.

Component

StepTimeline

Done / now / next step list for live, in-scenario advice, each step a single checkable action.

Component

RecapStat

Glass stat tile with a count-up numeral, used in a 2×2 grid on the Recap screen.

Component

RankingPodium

Top-three podium with gradient bars, followed by a plain list for the rest of the field.

Component · from v1

DebateAgentCard

One row per agent in the live debate: avatar, name, status, and either a shimmer skeleton or the landed analysis text.

Component · from v1

ResultCard

Attraction Gauge (a GradientMeter instance) plus subtext read, best response, and two alternative-response chips.

Component · from v1

ContactRow

Saved-contact list item: avatar, name, and a read-count caption that turns loss-averse at 2+ reads.

Component · from v1

ShareCard

Watermarked, story-shaped export card plus a single share action — summary only, never the raw conversation.

Target stack per brief: Next.js + Tailwind CSS + Framer Motion, replacing the Expo/React Native app going forward. Playbook, Confidence Meter, Recap, and Ranking have no backend yet — implementation will start against a typed mock api.ts. Debate reveal, Result, Saved Contacts, and Story share do have a real API already (the existing FastAPI swarm backend, unchanged) — those four should wire to it directly rather than mock it. This is a pure design pass — final feature-to-nav mapping (e.g. where the screenshot-upload entry point lives relative to the Live tab) is still a separate decision.
